## 2. Описание моей работы

В этом проекте я поработал с классами, изначально выбранным командой паттерном. На основе базового компонента DOMHelper сделал компонент кнопки для переиспользования.
Сделал на личном VDS простой бекенд на express/nodejs, просто ендпоинты для обращения и получения данных. Бекенд для типов задания "тесты" и "задачи".
Сделал сложный компонет для 4х разных тем - "задачи", где пользователь может выбирать нужную задачу по теме, писать код в редакторе и запускать его, видя пройденные и не пройденные тесты.
Номер решенных задач сохраняются в firebase.
Я проанализировал различные роутеры и выбрал самый простой и понятный (Vanilla Router), и добавил его в проект. Добавил анимацию загрузки страниц и данных с бекенда.

## 3. Личные Feature Components (для презентации)

Ниже представлены два ключевых компонента, которые я готов детально разобрать на технической защите.

- ### A page for writing solution task
  Этот компонент является одним из трёх базовых компонентов в приложении помимо страницы тестов и ai режима, реализованных другими членами команды.
- **Реализация:** Я создал страницу для решения различных задач, позволяя писать код решения прямо в браузере.
- **Техническая ценность:** Мною реализована возможность написания кода и его тестирования прямо в браузере. Тесты для тестирования загружаются с бекенда. Запуск кода для каждой темы реализован отдельно, потому что некоторые темы, особенно асинхронные и структурные задачи требуют особого подхода при запуске и тестировнии. Интерфейс страницы реализуется как SPA на классовом компоненте, без квери селекторов и динамически меняется при изменении данных.

- ### Backend
  Я написал простой бекенд на express+nodejs
- **Реализация:** Компонент включает в себя ендпоинты для отправки данных из заранее заполненных json файлов.
- **Техническая ценность:** Мною реализован бекенд на личном VDS сервере. Я не использовал для хранения SQL базы данных, потому что при разработки фронтенда, я очень часто менял данные. И просто легче их хранить просто в json файлах. Как данные для тестов, так и задач. Настроил nginx и CORS, чтобы не было проблем с обращением к беку. Написал в одном основном бекенд файле app.js эндпоинты на получения нужных данных, а после попросил ИИ сделать HTML страницу для тестрования работы всех эндпоинтов, для будущего использования мной и командой при написании нашего приложения. Для запуска бекенда использовал pm2 для удобного запуска в фоне, вместо открытия нового терминала, и для удобого просмотра логов при остановке и поломки сервера.
